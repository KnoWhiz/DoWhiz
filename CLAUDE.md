## Architecture Summary (Current Code)

DoWhiz is a multi-channel digital employee platform.

Primary runtime split:
- `inbound_gateway` receives webhook/event ingress and enqueues ingestion envelopes.
- `rust_service` (worker) consumes queue items and executes scheduler/tasks.

Queue/storage behavior:
- Ingestion queue backend defaults to Postgres in code, but gateway requires `servicebus`.
- Raw payload storage defaults to Supabase; Azure Blob is recommended for gateway production flow.
- Scheduler/user/index state is Mongo-backed.
- Account/auth/billing data is stored in Supabase Postgres (`AccountStore`).

Channels in code:
- email, slack, discord, sms, telegram, wechat, whatsapp, google_docs/google_sheets/google_slides, bluebubbles.

## Runtime Configuration Policy (Source of Truth)

- Runtime services load **unprefixed** keys from `DoWhiz_service/.env`.
- VM `.env` is generated by merging secret sets:
  - staging: `ENV_COMMON + ENV_STAGING`
  - production: `ENV_COMMON + ENV_PROD`
- I will also point you the latest local path for `ENV_COMMON`; `ENV_STAGING`; and `ENV_PROD` (in local files named as `.env.common`, `.env.staging`, and `.env.prod` accordingly). For loading keys in prod VM we follow same logic as in `.github/workflows/CICD-production.yml`; for loading keys in staging VM we follow same logic as in `.github/workflows/CICD-staging.yml`.
- Runtime `.env` must not contain `STAGING_*` or `PROD_*` keys.
- `DEPLOY_TARGET` is optional and used for runtime policy decisions (for example `run_task` backend auto behavior), not for shell-side key remapping.

Notes on legacy aliases:
- Some ingestion/raw-payload paths also read `SCALE_OLIVER_*` aliases via code fallback logic.
- Do not rely on alias-only keys in new deployments; keep unprefixed keys authoritative.

## RunTask Behavior (Important)

- `run_task_module` supports `codex` and `claude` runners.
- Execution backend:
  - `RUN_TASK_EXECUTION_BACKEND=local|azure_aci|auto`
  - `auto` resolves to `azure_aci` when `DEPLOY_TARGET` is `staging` or `production`, otherwise local.
- In staging/production targets, local codex execution is blocked by code unless backend is explicit Azure ACI flow.
